## Visual Design Elements

### Color Scheme (in actual PDF)
- **Title Text**: Black (#000000)
- **Section Headers**: Black (#000000)
- **Task Titles (Active)**: Black (#000000)
- **Task Titles (Completed)**: Dark Gray (#666666)
- **Metadata Text**: Gray (#999999)
- **Notes Text**: Light Gray (#AAAAAA)
- **Statistics Box**: Light Blue Background (#F2F2FA)
- **Separator Lines**: Light Gray (#CCCCCC)

### Typography (in actual PDF)
- **Title**: SF Pro Bold 28pt
- **Subtitle**: SF Pro Regular 14pt
- **Section Headers**: SF Pro Bold 18pt
- **Subsection Headers**: SF Pro Bold 14pt
- **Project Headers**: SF Pro Bold 14pt
- **Task Titles**: SF Pro Bold 11pt
- **Task Metadata**: SF Pro Regular 10pt
- **Task Notes**: SF Pro Regular 9pt

### Visual Indicators
- **Active Tasks**: ○ (hollow circle)
- **Completed Tasks**: ✓ (checkmark)
- **Separator Lines**: Thin horizontal lines
- **Statistics Box**: Rounded rectangle with light fill

### Page Layout
- **Page Size**: US Letter (8.5" × 11")
- **Margins**: 0.7" (50pt) on all sides
- **Content Area**: 7.1" × 9.6" (512 × 692 points)
- **Line Spacing**: 1.2x for readability

## Content Organization

### Page 1: Overview
1. **Header Section**
   - Main title (large, bold)
   - Generation timestamp
   - Visual breathing room

2. **Statistics Section**
   - Highlighted box with 2×2 grid
   - Key metrics at a glance
   - Professional appearance

3. **Distribution Section**
   - Status breakdown with counts
   - Priority breakdown with counts
   - Bullet-point lists

### Pages 2+: Task Details
1. **Project Grouping**
   - Clear project headers
   - Separator lines for visual clarity
   - Tasks sorted by creation date

2. **Task Cards**
   - Status indicator (○ or ✓)
   - Task title (bold)
   - Metadata line (status, priority, context, due date)
   - Notes preview (truncated to 200 chars)
   - Consistent spacing

3. **Smart Pagination**
   - Projects don't split across pages when possible
   - Tasks keep metadata together
   - No orphaned headers
   - Professional page breaks

## Comparison with Other Formats

### PDF vs HTML
**PDF Advantages:**
- Single file, no dependencies
- Consistent appearance across platforms
- Print-ready formatting
- Smaller file size

**HTML Advantages:**
- Can be edited
- Interactive elements possible
- More styling flexibility

### PDF vs Markdown
**PDF Advantages:**
- Professional formatting
- Page breaks and layout
- Better for printing
- Better for sharing

**Markdown Advantages:**
- Editable in any text editor
- Can be re-imported
- Better for version control
- Smaller file size

### PDF vs CSV
**PDF Advantages:**
- Human-readable formatting
- Visual hierarchy
- Professional appearance
- Better for presentations

**CSV Advantages:**
- Machine-readable
- Import to spreadsheets
- Data analysis friendly
- Better for automation

## User Workflow

### Exporting Tasks to PDF

1. **Open Export View**
   - Click "File" > "Export..." or use keyboard shortcut

2. **Select Format**
   - Choose "PDF (Formatted Report)" from format dropdown

3. **Configure Options**
   - Include/exclude completed tasks
   - Filter by date range (optional)
   - Filter by projects (optional)
   - Filter by contexts (optional)

4. **Preview**
   - See count of tasks that will be exported
   - Review warnings if any

5. **Export**
   - Click "Export" button
   - Choose save location
   - Wait for progress (< 2 seconds for 100 tasks)

6. **Review**
   - PDF opens automatically
   - Review formatting and content
   - Print or share as needed
